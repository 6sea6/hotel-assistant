🤖 携程采集助手

  • 入口：点击顶部"采集助手"，选择模板并粘贴携程酒店详情页或酒店列表页链接后加入任务队列
  • 详情页采集：单个酒店详情页会沿用原采集链路，读取酒店、房型、价格、评分、地址、交通与地铁信息，并通过比较助手写入桥接层保存
  • 列表页采集：列表页会先展开候选酒店，再逐家进入详情页采集；列表页解析优先读取 HTML/embedded JSON，候选不足时复用采集浏览器登录态做 CDP 兜底
  • 前筛设置：采集助手"前筛设置"中可配置携程 URL 前筛：价格、星级、排序、免费取消、点评数、评分档位；未知携程筛选片段会保留，不会被程序清空
  • 本地过滤：本地只保留目标宾馆数和排除住宿类型，用于限制列表页候选；房型、价格、退改、窗户和人数规则仍在详情页采集后复核
  • 登录态：采集会复用应用内 Edge profile；首次使用或登录态失效时会打开可见 Edge 窗口，让你登录携程后继续
  • 批量写入：列表页或批量任务会逐家采集详情，全部完成后统一批量写回；结果区会显示本次最终写入几家宾馆、几种房型
  • 取消任务：取消采集会显示"已取消"，并撤销本次任务已经写回的数据，避免留下半完成结果
  • 采集结果：单酒店和批量采集结果均显示在任务面板中，可查看详细采集信息

🏨 宾馆管理

  • 添加宾馆：点击左侧"添加宾馆"按钮，填写宾馆信息后保存
  • 编辑宾馆：点击宾馆卡片上的"编辑"按钮，修改信息后保存
  • 补充交通信息：可填写最近地铁站名称、距离最近地铁站的距离与公交路线，地铁距离填 0 时界面会显示“无较近地铁站”
  • 原始房型：可单独记录平台原始展示房型名，卡片会显示在宾馆名称下方，便于区分标准房型和平台原文
  • 卡片删除：卡片视图中每个宾馆都带删除按钮，第一次点击进入黄色确认态，再点一次才会删除
  • 行式删除：行式视图请勾选左侧复选框后，使用右上角"删除选中"统一删除
  • 收藏宾馆：点击"收藏"按钮，宾馆会显示金色边框标识

📋 模板系统

  • 创建模板：点击"模板管理" → "新建模板"，设置常用的入住日期、目的地等信息
  • 模板列表：每个模板会显示目的地、入住日期、离店日期和房间人数，方便快速核对行程范围
  • 应用模板：添加宾馆时选择模板，自动填充预设信息
  • 模板关联：宾馆只会使用你显式选择或写入的模板，不再根据日期自动猜测模板

🔍 搜索筛选与排序

  • 宾馆名称：程序会自动汇总当前已有宾馆名称，直接选择即可筛选
  • 携程评分：筛选 4.0 分或 4.5 分以上的宾馆
  • 收藏状态：筛选已收藏或未收藏的宾馆
  • 公共交通时间：筛选 20/40/60/80/100 分钟以内的宾馆
  • 最近地铁站筛选：支持筛选无、300 米、500 米、1km、1.5km 以内的宾馆
  • 按模板筛选：显示使用特定模板的宾馆，适合按不同出行计划分别查看
  • 排序方式： 当前列表可选择好评优先、低价优先、高价优先、距离近优先；切换后会立即重排当前筛选后的宾馆列表
  • 清除筛选： 清空宾馆名称、携程评分、收藏状态、模板、公共交通时间和最近地铁站距离等筛选条件
  • 交通补充信息：详情中可查看最近地铁站名称、最近地铁站距离与公交路线，地铁距离为 0 时显示"无较近地铁站"
  • 列表信息展示：卡片视图会在宾馆名称下显示原始房型，地址固定在左侧保留可读宽度，网址固定从右侧信息列起始位置开始显示并保持单行省略；若填写了公交路线，会在卡片信息区独占整行显示；行式视图不显示地址和公交路线，只在名称下补充原始房型
  • 网址展示：卡片中网址显示在地址右侧独立列，默认只显示首行并自动省略，但点击后仍会直接打开原网页

📊 排序规则

  • 好评优先：按携程评分从高到低排列，缺失评分的宾馆会排在后面
  • 低价优先：按总价从低到高排列，缺失价格的宾馆会排在后面
  • 高价优先：按总价从高到低排列，缺失价格的宾馆会排在后面
  • 距离近优先：按到目的地的距离从近到远排列，缺失距离的宾馆会排在后面
  • 同分处理：当主要排序字段相同时，程序会优先参考更低总价，再保持原有顺序稳定
  • 排名编号：列表中的 #1、#2、#3 等编号会随当前排序方式和筛选条件实时更新，前三名显示突出徽章
  • 导出排名图片：导出排名图片会使用当前页面的筛选条件和排序方式

🎨 个性化与设置

  • 个性化：点击顶部“个性化”按钮，可切换天空蓝、甜柠青、小草绿、菠萝黄、橡木棕、云朵白、秋仪金、铁杆粉、葡萄紫、糖果彩十套主题，并管理应用图标
  • 采集偏好：可控制“3人模板时是否额外保留携程显示可住4人的房型”
  • 应用图标：支持在个性化面板中选择本地图片作为窗口图标；所选图片会自动复制到当前数据目录内，删除原图也不会让窗口图标失效，也可以一键恢复默认图标
  • 数据存储位置：查看和更改数据存储路径
  • 快捷链接：快速访问携程、飞猪官网
  • AI接口：在设置中选择供应商并填写自己的 API Key，用于采集结果诊断
  • 高德 API Key：可在设置中填写自己的高德 Key，用于采集时计算公共交通、最近地铁站和路线；留空时使用程序默认 Key
  • 开发者工具：按 F12 打开后会优先切到中文界面，便于调试页面结构和脚本

💾 数据管理

  • 导出数据：将所有宾馆、模板和设置导出为JSON文件备份；如果当前启用了自定义窗口图标，也会一起写进导出文件
  • 导入数据：从本应用自己导出的 JSON 恢复数据，可选择覆盖当前宾馆、模板和设置，或仅把宾馆与模板追加到现有数据中；追加模式会保留当前设置和窗口图标，并自动重排冲突 ID，避免模板关联串号；任一步失败时都会回滚
  • 导出排名图片：将当前排名列表导出为PNG图片；为避免文字重叠，图片中不显示宾馆地址和交通路线，总价与日均会合并到同一信息格上下显示，其余字段按每行三个信息格重排，最近地铁站以“距地铁”显示且仅展示距离

💡 使用技巧

  • 使用模板可以快速添加多个行程相似的宾馆
  • 手动添加或编辑宾馆时，模板只会按你当前表单里的选择保存，不会再被程序自动改绑
  • 收藏功能可以标记心仪的宾馆，方便对比
  • 列表右上角的刷新图标会重新拉取当前窗口内的宾馆、模板和设置数据，不需要重启应用
  • 需要批量删除时，先切到行式视图勾选多家宾馆，再使用右上角删除选中
  • 导出数据可以备份重要信息，防止数据丢失
  • 排名图片可以分享给朋友一起讨论
